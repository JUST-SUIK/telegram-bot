# ADR-001: Telegram Bot Agent 系统架构决策

> 状态：已批准
> 日期：2026-06-22
> 决策者：项目团队

---

## 背景

需要开发一个 Telegram Bot Agent，用于通过手机远程监控和操控电脑软件。核心需求包括：
- 进程状态监控
- 远程命令执行
- 截图和 GUI 自动化
- AI 辅助识别和操作
- 日志读取和分析

---

## 决策

### 1. 运行环境

**决策**：Windows 原生

**理由**：
- pyautogui 截图和 GUI 控制在 Windows 原生环境下更直接
- 无需 WSL 的额外配置和兼容性问题
- 更好的系统 API 访问能力

**备选方案**：
- WSL：Claude Code CLI 可以直接用，但 GUI 操作受限

---

### 2. AI 后端架构

**决策**：双 Agent 架构

**架构设计**：
```
┌─────────────────────────────────────────────────────────┐
│                  Telegram Bot Server                      │
│                                                          │
│  ┌─────────────────┐      ┌─────────────────────────┐  │
│  │ 主线程 Agent     │      │ 视觉识别 Agent           │  │
│  │ (mimo-v2.5-pro)  │      │ (mimo-v2.5 全模态)      │  │
│  │ - 命令执行       │      │ - 截图分析               │  │
│  │ - 代码问答       │      │ - 按钮识别               │  │
│  │ - 任务管理       │      │ - 坐标定位               │  │
│  └─────────────────┘      └─────────────────────────┘  │
│           │                         │                    │
│           └────────┬────────────────┘                    │
│                    ▼                                     │
│            共享状态管理                                    │
└─────────────────────────────────────────────────────────┘
```

**模型配置**：
- 主线程 Agent：`mimo-v2.5-pro`（LLM，文字/编码工作）
- 视觉识别 Agent：`mimo-v2.5`（全模态，截图分析、元素识别）

**API 配置**：
- 兼容 OpenAI 接口协议：`https://token-plan-cn.xiaomimimo.com/v1`
- 兼容 Anthropic 接口协议：`https://token-plan-cn.xiaomimimo.com/anthropic`
- API Key：从 `.env` 文件读取

**理由**：
- 可以同时运行两个 Agent，互不干扰
- 文字/编码任务使用更智能的 LLM
- 视觉任务使用全模态模型
- 无需切换模型，提高效率

---

### 3. 截图方案

**决策**：mss

**理由**：
- 速度比 PIL 快 2-3 倍
- 支持多显示器
- 跨平台兼容性好
- 可以截取指定区域

**备选方案**：
- PIL.ImageGrab：简单，但多显示器支持差
- pyautogui：和 GUI 自动化统一，但速度一般

---

### 4. GUI 自动化方案

**决策**：pyautogui + AI 辅助识别

**工作流程**：
```
用户：/click 确认按钮
Bot：📸 正在截图分析...
     🔍 AI 识别中：找到"确认"按钮在坐标 (450, 320)
     ⚠️ 将点击确认按钮，是否继续？
用户：确认
Bot：✅ 已点击确认按钮
```

**实现方式**：
1. 使用 mss 截取当前屏幕
2. 使用视觉识别 Agent（mimo-v2.5）分析截图
3. 识别目标元素（按钮、输入框等）的坐标
4. 使用 pyautogui 执行点击/输入操作

**优点**：
- 不需要用户知道坐标
- 可以识别按钮、输入框、菜单等
- 适应不同分辨率和窗口位置

### 5. 代理配置

**决策**：Clash Verge

**配置**：
- 代理地址：`127.0.0.1:7897`
- 代理类型：SOCKS5/HTTP

**理由**：
- 国内访问 Telegram 需要代理
- Clash Verge 是常用的代理软件

---

### 6. 安全配置

**决策**：用户白名单 + 命令黑名单 + 二次确认

**用户白名单**：
- Telegram 用户 ID：`6531095340`
- 只允许指定用户操作

**命令黑名单**：
- `rm -rf`：删除所有文件
- `format`：格式化磁盘
- `shutdown`：关机
- `del /f /s /q`：Windows 强制删除
- `taskkill /f`：强制结束进程
- 其他危险命令...

**二次确认机制**：
- 所有 `/exec` 命令执行前需要用户确认
- GUI 点击操作执行前需要用户确认
- 危险操作会告知风险和禁止原因

**示例**：
```
用户：/exec rm -rf /
Bot：❌ 命令被拒绝
     命令：rm -rf /
     风险：此命令会删除系统所有文件，导致数据丢失
     此命令已被列入黑名单，禁止执行
```

---

### 7. 监控配置

**决策**：进程状态监控 + CPU/内存监控

**监控内容**：
- 进程是否在运行
- 进程是否异常退出（退出码）
- CPU 使用率
- 内存使用率
- 日志输出

**监控间隔**：60 秒

**汇报机制**：
- 定时汇报：每 60 秒自动汇报任务进度
- 即时汇报：用户命令 `/status` 时立即汇报

---

### 8. 日志配置

**决策**：项目根目录下的 log 目录

**配置**：
- 日志路径：`e:/Project/telegram-bot-agent/log/`
- 日志文件：`bot.log`
- 日志级别：INFO
- 日志轮转：最大 10MB，保留 5 个备份

### 10. 启动方式

**决策**：打包成 exe

**理由**：
- 无需 Python 环境
- 双击即可运行
- 方便分享和使用

**打包工具**：PyInstaller 或 cx_Freeze

**预估大小**：约 50-100MB

---

### 11. `/ask` 命令配置

**决策**：调用 Claude Code CLI

**工作目录**：`E:/Project`

**命令格式**：
- `/ask <问题>`：在默认工作目录执行
- `/ask --dir <目录> <问题>`：在指定目录执行

**用途**：
- 代码审查和 bug 分析
- 技术问题解答
- 代码重构建议
- 文件操作（读取、修改代码）
